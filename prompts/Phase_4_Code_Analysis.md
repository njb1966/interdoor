# Phase 4 Code Analysis Prompt

## Task

Analyze existing InterDoor or legacy game code before changing it.

## Required Context

Read:

- `01_Project_Rules.md`
- `06_Code_Modernization.md`
- `07_Backlog.md`

## Analysis Checklist

- What is the current architecture?
- What is framework code and what is game code?
- What state is local, broadcast, snapshot, or shared reference?
- What behavior is already tested?
- What behavior is only documented?
- What docs conflict with current code?
- What is the smallest safe change?

## Output

Provide findings first, with file references. Then recommend a minimal implementation path.
